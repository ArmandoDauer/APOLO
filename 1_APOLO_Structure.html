APOLO Structure

Designing a DRT system is complex due to many configuration options and the need to align them with specific goals. To support this process, APOLO was created as a structured framework to guide decision-makers.

APOLO works through a series of tools that, using user inputs, help answer: “What is the most effective and efficient DRT setup at a strategic level?”

Using inputs like Directive Variables, user flexibility perception, simulation data, and cost-benefit preferences, APOLO identifies a suitable DRT configuration (Operational and Service Variables).

To handle the complexity of DRT design, APOLO is divided into three main modules:

  • MUSE: Suggests suitable values for Operational Variables based on user goals, flexibility needs, and past DRT cases.
  • ARTEMIS: Simulates the proposed system to assess performance using Key Performance Indicators (KPIs).
  • DELOS: Helps choose Service Variable values through a cost-benefit analysis of their impacts.

The process starts with defining Directive Variables, which reflect high-level goals and help shape the system design.

While a DRT can serve more than one purpose, it’s best to focus on one or two to avoid conflicting requirements.

MUSE then uses a knowledge-based matrix to suggest how often certain Operational Variable values appear in DRTs with similar goals.

User perception of flexibility is considered to adjust these suggestions to local needs, balancing service quality and resource use.

Based on these insights, MUSE classifies each Operational Variable value as “Marginally Suitable,” “Suitable,” or “Highly Suitable.” Decision-makers select values accordingly.

Next, ARTEMIS simulates system performance using realistic demand and setup parameters, producing KPIs to evaluate efficiency and identify improvements.

If KPIs are acceptable, the process moves to DELOS; if not, the setup can be adjusted and retested. APOLO encourages iteration to improve the design.

DELOS analyzes the pros and cons of each Service Variable value. Decision-makers rate the impacts from “Strong Cost” to “Strong Benefit,” and values are grouped from “Mostly Costly” to “Mostly Beneficial.”

Finally, the full system setup is reviewed to check if it meets strategic goals. If not, the process loops back to improve the configuration.

APOLO provides guidance based on real DRT experience, but final decisions rest with local authorities, operators, and users through iterative design and review.
